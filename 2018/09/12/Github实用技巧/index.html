Github实用技巧

发表于2018-09-12|更新于2021-06-18|实用技巧
|字数总计:746|阅读时长:2分钟|阅读量:

Github有很多优秀的开源框架，各种技术大佬混迹其中，其中有他们总结的学习教程，造好的轮子（开发的各种工具，技术框架），开源的项目等。
我们能够从中借鉴学习，也可以获取灵感，或节省时间，完成任务等等。但是我们要如何从如此多的内容中找到优秀的和我们需要的呢？

背景介绍

还在为自学时找不到适合练手的项目而苦恼？还在好奇别人是如何在GitHub众多项目中找到高质量代码的？真的是因为他们独具慧眼吗？不，其实他们只是掌握了正确的搜索方法。接下来本文就来介绍一些常用的GitHub高级搜索方法，
相信你看完之后也能很快在GitHub众多项目中找到自己所需的项目。

动手实践

1. 根据star，fork数搜索

当我们想在github上查找一个使用springboot的优秀项目的时候，评价其优秀与热门最直接的参数就是看这个项目被star（点赞）了多少次，被fork（收录）了多少次。虽然一个优秀的项目不一定是star多，有可能暂时还没被其他人给发现，但根据这2个参数筛选却是我们简单有效的依据。

下面我们来动手：

  • 语法一： 关键字 stars:>= 数量 forks:>=数量 (可单独搜索，也可组合搜索，中间用空格隔开)

搜索springboot 的star>=5000，且fork数>=5000的相关内容

总结

我们想要进行精准搜索，无非就是增加筛选条件。

  • in:name xxx 按照项目名搜索
  • in:readme xxx 按照README搜索
  • in:description xxx 按照description搜索
  • stars:>xxx stars数大于xxx
  • forks:>xxx forks数大于xxx
  • language:xxx 编程语言是xxx
  • pushed:>YYYY-MM-DD 最后更新时间大于YYYY-MM-DD

以上就是在GitHub上面精准搜索项目的一些小技巧，在使用时可以混合使用，希望能帮到你！😏
