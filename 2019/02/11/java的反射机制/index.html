java的反射机制

发表于2019-02-11|更新于2021-06-18|语言特性反射
|字数总计:3.2k|阅读时长:14分钟|阅读量:

今天我们来了解Java语言的特性之一，反射机制（Reflection）。反射是框架设计的灵魂，为什么这么说呢？接下来请跟我一点一点地来剖析它!

概述

java反射机制指的是在java程序运行过程中，对于任意的类都可以获取它的所有属性以及方法，对于任意一个对象都可以任意的调用他的属性和方法，这种动态获取对象信息和动态调用对象方法的功能称为java反射机制。

万事万物都有两面性，有反，当然肯定是有正的。要想更好的了解反射机制，我们不妨通过正常的Java运行过程来倒推它的原理，这里我采用图解的方式来呈现。接下来就是两个字：

安排！

由上图可知在程序运行时，当需要生成一个Person类的实例对象时，JVM就会检查此类是否已经装载到内存中。若是没有装载，则把.class文件装入到内存中（也就是生成class对象）。若是装载，则根据class对象直接生成实例对象。

PS：阅读时请不要混淆class对象和实例对象，一个类的class对象始终只能存在一个，里面存放的是类的信息，实例对象可以存在多个！

反射机制的本质理解就是得到类的class对象后，反向获取类对象的各种信息。

作用：
通过反编译.class ------------->.java 来访问java对象的属性，方法，构造方法等。
