RedisSerializer 的实现类主要有以下几种：

主要分成五类：

JDK 序列化方式

JdkSerializationRedisSerializer:

RedisTemplate默认序列化。
优点：
1. 通用性强
2. 反序列化时不需要提供类型信息(class)
缺点：
1. 序列化速度慢
2. 序列化占用内存大
3. 序列化对象必须实现Serializable接口
4. 可读性差

String 序列化方式

StringRedisSerializer:

StringRedisTemplate默认序列化。
优点：
1. 可读性强
2. 不需要转换
缺点：
1. 只能对字符串序列化
2. 不能对对象序列化

GenericToStringSerializer:

同StringRedisSerializer一样，但它可以将任何对象泛化为字符串并序列化。(基本不会去使用)

JSON 序列化方式

Jackson2JsonRedisSerializer:

使用Jackson库将对象序列化为JSON字符串。
优点：
1. 速度快
2. 序列化后的字符串短小精悍
3. 不需要实现Serializable接口
缺点：
1. 必须提供要序列化对象的类型信息(.class对象)

FastJsonRedisSerializer:

由阿里巴巴FastJson包提供
优点：
1. 速度快
2. 兼容性强
3. 占用内存小
缺点：未知

GenericFastJsonRedisSerializer:

同FastJsonRedisSerializer类似

GenericJackson2JsonRedisSerializer:

底层使用Jackson进行序列化并存入Redis。
优点：
1. 对于普通类型(如数值类型，字符串)可以正常反序列化回相应对象
缺点：
1. 存入对象时由于没有存入类信息，则无法反序列化

XML 序列化方式

OxmSerializer:

以xml格式存储（但还是String类型），解析起来比较为复杂。
缺点:
1. 速度慢
2. 占用空间大

ByteArray 序列化方式

ByteArrayRedisSerializer:

Byte数组序列化
